# CLAUDE.md — Regras do projeto nexform

## Fluxo obrigatório para toda alteração

1. **Criar branch nova** com nome descritivo (ex: `feat/menu-hamburger`, `fix/modal-responsividade`)
2. **Fazer as alterações** e commitar com mensagem clara
3. **Criar PR** para revisão após o push — SEMPRE, sem exceção

Nunca fazer commit direto na `main`. Nunca reaproveitar branch de outra feature para uma alteração diferente.
